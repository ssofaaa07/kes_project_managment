## ФАЗА 3. Оценка проекта

## 1. Сценарии использования системы 

Акторы системы: Пользователь, Организатор, Участник, Внешний календарь, Email-сервис

### UC-1. Создать разовую встречу
*Акторы*: Организатор, Email-сервис\
*Цель*: создать единичную (не повторяющуюся) встречу с деталями и участниками\
*Предусловия*: Пользователь авторизован в системе\
*Основной поток*:
1. Организатор нажимает кнопку «Создать встречу»
2. Организатор заполняет тему, описание, дату, время, продолжительность
3. Организатор добавляет участников
4. Организатор нажимает кнопку «Создать»
5. Система создаёт встречу и рассылает уведомления участникам


*Негативные*:
1. Конфликт времени у участников\
После нажатия кнопки «Создать» Система выводит информационное окно с указанием участника и промежуток времени, с которым возникло пересечение. Организатор может:
  +	вернуться на экран редактирования деталей встречи по кнопке «Редактировать», 
  +	отменить создание встречи по кнопке «Отменить»
  +	создать встречу по кнопке «Создать»

*Постусловия*: У организатора и участников в календаре отображается созданная встреча

### UC-2. Создать повторяющуюся встречу
*Акторы*: Организатор, Email-сервис\
*Цель*: создать встречу, повторяющуюся по определённому расписанию\
*Предусловия*: Пользователь авторизован в системе\
*Основной поток*:
1) Организатор нажимает кнопку «Создать встречу»
2) Организатор заполняет детали встречи
3) Организатор отмечает опцию «Повторяющаяся встреча»
4) Организатор добавляет участников
5) Организатор нажимает кнопку «Создать»
6) Система создаёт встречу и рассылает уведомления участникам


*Альтернативные*:
1) Настройка исключений\
– Организатор отмечает в календаре (перечисляет) даты, в которые создаваемая встреча не будет назначена


*Негативные*:
1) Слишком длинная серия\
– В системе введено ограничение на максимальное количество встреч – 50

*Постусловия*: У организатора и участников в календаре отображается созданная встреча

### UC-3. Принять/Отклонить приглашение
*Акторы*: Участник\
*Цель*: подтвердить участие во встрече\
*Предусловия*: Участник получает уведомление о приглашении на встречу\
*Основной поток*:
1) Участник получает уведомление о создании встречи
2) Участник открывает приглашение
3) Участник нажимает кнопку «Принять»
4) Система добавляет встречу в календарь Участника

*Альтернативные*:
1) Предложить другое время\
– Участник нажимает кнопку «Предложить другое время», выбирает время и отправляет всем участником письмо с предложением.

*Негативные*:
1) Конфликт с существующими встречами\
– В случае конфликта со встречами Участник видит предупреждение.

*Постусловия*: В информации о встрече участник отображается как подтвердивший своё участие

### UC-4. Перенести встречу
*Акторы*: Организатор, Email-сервис\
*Цель*: изменить время существующей встречи\
*Предусловия*: У Организатора есть запланированные встречи\
*Основной поток*:
1) Организатор открывает встречу
2) Организатор нажимает кнопку «Перенести»
3) Организатор выбирает новое время и нажимает кнопку «Сохранить»
4) Система обновляет данные встречи
5) Система уведомляет участников

*Негативные*:
1) Конфликт времени у участников\
– После нажатия кнопки «Создать» Система выводит информационное окно с указанием участника и промежуток времени, с которым возникло пересечение. Организатор может:
•	вернуться на экран редактирования деталей встречи по кнопке «Редактировать», 
•	отменить создание встречи по кнопке «Отменить»
•	создать встречу по кнопке «Создать»

*Постусловия*: Время проведения встречи изменено

### UC-5. Отменить встречу
*Акторы*: Организатор, Email-сервис\
*Цель*: отменить запланированную встречу\
*Предусловия*: У Организатора есть запланированные встречи\
*Основной поток*:
1) Организатор открывает встречу
2) Организатор нажимает кнопку «Отменить»
3) Организатор подтверждает «Отмену»
4) Система удаляет встречу
5) Система уведомляет участников

*Постусловия*: Встреча удалена из календарей Организатора и Участников

### UC-6. Настроить уведомления и напоминания
*Акторы*: Пользователь\
*Цель*: настроить параметры оповещений\
*Предусловия*: Пользователь авторизован\
*Основной поток*:
1) Пользователь заходит в Настройки своего профиля, раздел Уведомления
2) Пользователь выбирает способы получения уведомлений: email, SMS, push (один или несколько)
3) Пользователь устанавливает время дополнительных напоминаний (или отключает их)
4) Пользователь нажимает кнопку «Сохранить»
5) Система выполняет сохранение настроек

*Постусловия*: Настройки напоминаний изменены

### UC-7. Синхронизировать с внешним календарем
*Акторы*: Пользователь, Внешний календарь\
*Цель*: импортировать/экспортировать встречи в другие календари\
*Предусловия*: Пользователь авторизован в системе и имеет учётную запись в стороннем сервисе\
*Основной поток*:
1) Пользователь заходит в Настройки своего профиля, раздел Синхронизация
2) Пользователь выбирает систему (Google календарь, Outlook)
3) Пользователь авторизуется через Систему авторизации внешнего календаря
4) Пользователь настраивает направление и параметры синхронизации
5) Система синхронизирует встречи

*Постусловия*: Установлено соединение с внешним сервисом, настроена синхронизация календаря

### UC-8. Создать шаблон встречи
*Акторы*: Организатор\
*Цель*: сохранить настройки встречи для многократного использования\
*Предусловия*: Пользователь авторизован в системе\
*Основной поток*:
1) Организатор нажимает кнопку «Создать встречу»
2) Система отображает окно создания встречи
3) Организатор вводит данные встречи
4) Организатор нажимает кнопку «Сохранить как шаблон»
5) Организатор указывает название шаблона
6) Система сохраняет шаблон встречи

*Постусловия*: Сохранён шаблон, который Организатор может использовать при создании новой встречи

### UC-9. Просмотреть метрики участия
*Акторы*: Организатор\
*Цель*: анализировать активность и посещаемость участников\
*Предусловия*: Пользователь авторизован в системе\
*Основной поток*:
1) Организатор заходит в раздел «Аналитика»
2) Организатор выбирает период для анализа
3) Система выводит данные: процент принятых/отклонённых приглашений, самых активных участников, статистику по времени встреч
4) Организатор нажимает кнопку «Экспорт»
5) Система выполняет формирование и загрузку отчёта

*Альтернативные*:
1) Пользователь ранее не организовывал встречи

*Негативные*:
1) Выбор некорректного периода для формирования отчётности

*Постусловия*: Пользователь получает аналитический отчёт о проведённых встречах

### UC-10. Делегировать организацию встречи
*Акторы*: Организатор, Email-сервис\
*Цель*: передать права организатора другому участнику\
*Предусловия*: У Организатора есть запланированные встречи\
*Основной поток*:
1) Организатор открывает встречу
2) Организатор нажимает кнопку «Делегировать организатора»
3) Организатор выбирает участника из списка
4) Организатор подтверждает передачу прав
5) Система изменяет данные об организаторе встречи и уведомляет участников

*Постусловия*: Организатор встречи изменён

### UC-11. Управление доступом к календарю
*Акторы*: Пользователь\
*Цель*: настроить видимость календаря для других пользователей\
*Предусловия*: Пользователь авторизован\
*Основной поток*:
1) Пользователь заходит в Настройки
2) Пользователь выбирает уровень доступа из списка. Варианты видимости календаря: полная видимость (весь календарь), только активность (занят/свободен), для некоторых коллег, полная приватность.
3) Пользователь нажимает на кнопку «Сохранить»
4) Система выполняет сохранение настроек пользователя

*Альтернативные*:
1) Выбор варианта «для некоторых коллег»\
– Пользователь вводит список коллег, которым он предоставляет доступ к своему календарю.

*Постусловия*: Изменены настройки доступа к календарю Пользователя
